# Проект по компрессии аудиоданных с использованием метода главных компонент (PCA)

Репозиторий содержит практическое исследование применения математических методов линейной алгебры и машинного обучения для сжатия звуковых сигналов формата WAV без использования специализированных аудиокодеков.

## Описание проекта
**Файл:** `PCA.ipynb`

Цель проекта — реализовать алгоритм сжатия звуковой дорожки (на примере фрагмента Сонаты для скрипки №10 Бетховена) с помощью уменьшения размерности матрицы признаков методом главных компонент (PCA). В работе исследуется математическая зависимость между количеством главных компонент, качеством восстановленного аудиосигнала и итоговой степенью сжатия файла.

### Основные этапы исследования:
* **Загрузка и первичный анализ (EDA):** Импорт аудиофайла при помощи библиотеки `scipy.io.wavfile`.
  * Анализ частоты дискретизации (стандартные 44100 Гц) и структуры каналов (стереосигнал).
  * Визуализация амплитудно-временных графиков для левого и правого каналов звука.
* **Предобработка данных:** Подготовка матрицы амплитуд звуковых волн для корректной подачи на вход алгоритму PCA.
* **Применение PCA и понижение размерности:**
  * Реализация автоматического подбора оптимального числа компонент с помощью встроенных алгоритмов (`n_components='mle'`) и через задание порога объясненной дисперсии (от 0 до 1).
  * Анализ собственных векторов матрицы для выделения наиболее значимых звуковых признаков.
* **Реконструкция и фильтрация шумов:** * Обратное преобразование данных из пространства главных компонент в исходную форму сигнала.
  * Оценка субъективного качества звучания с помощью `IPython.display.Audio`.
  * Очистка аудио от фонового шума путем отсечения низкоранговых компонент, отвечающих за высокочастотные помехи.

### Ключевые выводы
* Метод главных компонент (PCA) успешно применим для задач компрессии raw-аудиосигналов, позволяя регулировать баланс между размером файла и качеством звука.
* Установлена прямая математическая связь: степень сжатия звука эквивалентна отношению выбранного числа новых компонент к их начальному количеству.
* Алгоритм позволяет не только сжимать данные, но и эффективно фильтровать фоновые шумы при условии тонкой настройки количества оставляемых компонент (чтобы избежать эффекта неестественного "бочкового" звучания).

## Используемые технологии
Проект реализован на языке Python с использованием следующего стека:
* **NumPy** — матричные вычисления, манипуляции со структурой аудиосигналов.
* **SciPy (`scipy.io.wavfile`)** — чтение и десериализация аудиофайлов формата `.wav`.
* **Scikit-learn (`sklearn.decomposition.PCA`)** — реализация математического алгоритма метода главных компонент.
* **Matplotlib** — построение графиков волновых форм (waveform) и визуализации результатов фильтрации.
